# DeskTransfer 使用指南

## 简介
DeskTransfer 是一个简单的局域网图片传输工具，可以在同一局域网内的设备之间快速传输图片文件。

## 快速开始

### 1. 启动应用程序

#### 方法一：使用启动脚本
- **Windows**: 双击运行 `start.bat`
- **Linux/macOS**: 在终端中运行 `./start.sh`

#### 方法二：直接运行
- **接收端**: `python receiver.py`
- **发送端**: `python sender.py`

### 2. 使用步骤

1. **启动接收端**
   - 运行接收端应用程序
   - 点击"启动服务器"按钮
   - 记录显示的IP地址和端口号（默认为12345）

2. **启动发送端**
   - 运行发送端应用程序
   - 在"接收端IP地址"中输入接收端的IP地址
   - 端口号保持默认（12345）

3. **选择并传输文件**
   - 点击"选择文件"按钮，选择要传输的图片文件
   - 可以选择多个图片文件（支持批量传输）
   - 点击"连接"按钮连接到接收端
   - 连接成功后，点击"发送文件"开始传输

4. **监控传输进度**
   - 发送端会显示传输进度
   - 接收端会显示接收进度和文件保存位置

## 功能特点

- **批量传输**: 支持一次选择和传输多个图片文件
- **实时进度**: 显示传输进度和速度
- **多种格式**: 支持 JPG、PNG、GIF、BMP、TIFF、WebP 等常见图片格式
- **简单易用**: 直观的图形界面，操作简单
- **跨平台**: 支持 Windows、Linux 和 macOS

## 支持的图片格式

- JPEG (.jpg, .jpeg)
- PNG (.png)
- GIF (.gif)
- BMP (.bmp)
- TIFF (.tiff)
- WebP (.webp)

## 常见问题

### Q: 无法连接到接收端？
A: 请检查以下几点：
1. 确保两台设备在同一局域网内
2. 确保接收端已启动服务器
3. 检查IP地址和端口号是否正确
4. 检查防火墙设置

### Q: 传输速度慢？
A: 传输速度受以下因素影响：
1. 网络带宽
2. 文件大小
3. 网络质量

### Q: 支持传输其他类型的文件吗？
A: 当前版本仅支持图片文件传输，后续版本可能会支持更多文件类型。

## 技术细节

- **通信协议**: 基于TCP的自定义协议
- **端口**: 默认使用12345端口
- **缓冲区大小**: 4KB
- **文件保存位置**: `data/received/` 目录

## 开发信息

- **开发语言**: Python
- **GUI框架**: Tkinter
- **网络库**: Socket
- **图像处理**: Pillow
